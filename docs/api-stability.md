# API 안정성 정책

SkidMC는 API가 제공하는 정보·기능의 지원과 그 정보에 접근하는 코드 계약의 호환성을
구분한다. `stable`과 `unstable`의 차이는 기능의 정확성이나 지원 수준이 아니라 접근
계약의 호환성 범위다.

## Stable API

`unstable` 하위 패키지에 속하지 않은 공개 API는 stable API다.

- 패치 버전에서는 기존 소스 및 바이너리 호환성을 유지한다.
- 마이너 버전에서는 호환성을 유지하면서 기능을 추가할 수 있다.
- 호환성을 깨는 변경은 메이저 버전에서만 진행한다.
- 잘못된 동작을 수정하는 과정에서 결과가 달라질 수 있지만, 기존 계약을 불필요하게
  변경하지 않는다.

## Unstable API

경로에 `.unstable.`이 포함된 공개 API는 unstable API다. Unstable API가 제공하는
정보와 기능은 stable API와 같은 수준으로 지속적으로 지원하지만, 타입, 함수,
프로퍼티, 패키지와 같은 접근 계약의 호환성은 보장하지 않는다.

- 패치 버전에서 다른 unstable API로 변경될 수 있다.
- 마이너 버전에서 대응하는 stable API로 이전될 수 있다.
- 변경 시 같은 정보나 기능에 접근할 수 있는 대체 경로를 함께 제공하고 변경 사항을
  릴리스 노트에 명시한다.
- Unstable API를 사용하는 모드는 SkidMC를 업데이트할 때 `.unstable.` import를
  검색하고 변경 사항을 확인해야 한다.

Unstable API에는 다음 두 종류가 있다.

### 임시 Unstable API

지원할 정보나 기능은 확정됐지만 장기적인 소유 타입과 수명 주기 같은 안정 계약이 아직
준비되지 않은 API다. 대응하는 stable API가 준비되면 그 계약으로 이전될 수 있다.

`Kart.currentLap`과 `Kart.maxLap`은 레이스 상태 API가 마련될 때까지 제공하는 임시
unstable API다. 랩 정보의 제공은 계속 지원되지만 접근 위치와 반환 계약은 변경될 수
있다.

### 의도적으로 유지되는 Unstable API

외부 데이터팩의 구현 세부사항처럼 SkidMC가 호환성을 통제할 수 없는 구조를 직접
노출하는 API다. 이러한 API는 stable 승격을 목표로 하지 않으며 계속 unstable 상태로
유지될 수 있다.

`KnownAttrModId`와 modifier별 이벤트는 마크라이더 데이터팩의 attribute modifier
구조를 반영한다. 카트 상태 정보의 제공은 계속 지원하지만 개별 ID, 상수 이름, 이벤트
구성은 데이터팩 변경에 맞춰 호환성 없이 바뀔 수 있다.

## 호환성 요약

| 보장 대상 | Stable API | Unstable API |
| --- | --- | --- |
| 정보·기능의 지속적인 지원 | 보장 | 보장 |
| 값과 동작의 정확성 | 보장 | 보장 |
| 패치 버전 접근 계약 호환성 | 보장 | 보장하지 않음 |
| 마이너 버전 접근 계약 호환성 | 보장 | 보장하지 않음 |
| stable API로의 이전 | 해당 없음 | 가능하지만 필수는 아님 |

## 프리릴리스

`alpha`, `beta`, `rc`와 같은 프리릴리스에서는 stable API도 최종 1.0.0 계약을 확정하기
위해 변경될 수 있다. 이러한 변경도 릴리스 노트에 기록하고 가능한 경우 대체 경로를
제공한다. 정식 1.0.0 이후에는 위의 stable 호환성 규칙을 적용한다.
